# PR 描述规范

当你修改了一个功能（例如按钮逻辑）时，请在 PR 描述中清晰说明以下内容：

## 为什么要改

说明修改原因——是修复 **BUG** 还是基于 **需求**。请简要描述背景，并附上 **JIRA 链接**。

**示例：**

修复 KAT-1234 中描述的按钮点击无效问题。原因为事件绑定错误，导致状态未更新。

## 你做了什么

具体说明你在代码层面做了哪些修改，例如：

+ 调整了逻辑判断条件
+ 修改了组件状态管理
+ 增加了接口请求或参数传递

**示例：**

调整了 ButtonAction 组件逻辑，增加点击防抖，修复多次触发问题。

## 影响范围

说明该功能或按钮被使用的位置，以及可能影响到的其他页面或模块。

**示例：**

此按钮同时在订单详情页和用户信息页中使用，修改后需验证这两个页面的行为是否正常。

## 如何测试

详细说明测试步骤，尤其是当功能需要满足特定条件才能展示或触发时（例如按钮需处于某种状态才可见）。

请附上 **截图或录屏**，以证明你已验证过相关功能。

**示例：**

1. 登录测试账号
2. 进入订单详情页
3. 点击"重新支付"按钮
4. 验证按钮状态变化及支付流程是否正常
